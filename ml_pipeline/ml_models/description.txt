#   -> baseModel.py определяет интерфейс для всех моделей.
#   -> modelFactory.py позволяет создавать различные модели.
#   -> modelRegistry.py управляет всеми моделями централизованно.
#   -> lstmWithAttentionModel/ содержит код для конкретной модели LSTM с Attention.


Как это работает вместе?

Модели создаются через фабрику:
При запросе модели (например, lstm_attention) через ModelFactory,
создается объект соответствующей модели, например, LSTMWithAttentionModel.

Модели регистрируются в реестре: После создания модели, ее можно зарегистрировать в ModelRegistry,
чтобы впоследствии извлечь и использовать для предсказаний.

Модели обучаются и используются: Все модели имеют общий интерфейс с методами train(), predict(), save() и load(),
что позволяет одинаково работать с разными типами моделей в вашем пайплайне.


Резюме:
Структура позволяет вам легко расширять систему:

baseModel.py определяет интерфейс для всех моделей.
modelFactory.py позволяет создавать различные модели.
modelRegistry.py управляет всеми моделями централизованно.
lstmWithAttentionModel/ содержит код для конкретной модели LSTM с Attention.
